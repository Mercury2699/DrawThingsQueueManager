# Draw Things Queue Manager

A sleek, modern, glassmorphic web dashboard and queue processor for the macOS **Draw Things** AI image generation app. 

This tool provides the queue management capabilities missing from the core Draw Things app, allowing you to queue up prompts, compare them across different base models, schedule batch runs, reorder tasks, and catalog outputs inside a unified gallery.

---

## Key Features

- **🚀 Real-Time Generation Queue**: Automatically queues prompts and runs them sequentially.
- **🎭 Base Model Comparison**: Add a prompt once, select multiple base models, and the system automatically generates versions for each model.
- **🎛️ Active LoRA Weights Control**: Choose multiple LoRAs and adjust their generation weights directly using interactive sliders.
- **✨ Drag-and-Drop Reordering**: Smooth drag-and-drop tasks to prioritize them, or use simple up/down controls.
- **🖼️ Image Gallery & Parameters Reuse**: All successfully generated images are saved with their parameters. Click any image to view details or click **"Reuse Parameters"** to populate the generator form instantly.
- **🔍 Local Directory Scanning**: Automatically scans your macOS Draw Things Models directory to list your downloaded checkpoint models and LoRA files.

## Quickstart

### Prerequisites

1. **Draw Things App**: Ensure Draw Things is running on your Mac.
2. **API Server Enabled**: In Draw Things, go to **Settings** -> Scroll to **API Server** -> Turn it **ON** (default port `7860`).
3. **Python 3** and **uv** (fast Python runner).

### Running the Manager

From the root directory of this repository, run:

```bash
uv run queue_manager.py
```

This will automatically resolve dependencies (FastAPI, Uvicorn, Pillow, Requests) inside an isolated environment and boot the server at:

👉 **[http://localhost:8000](http://localhost:8000)**

## File Structure

- `queue_manager.py`: FastAPI server, SQLite database initializations, background worker loop, and API endpoints.
- `static/`: Frontend single-page application.
  - `index.html`: Web interface skeleton using semantic HTML and inline SVGs.
  - `style.css`: Sleek glassmorphic dark-mode stylesheets.
  - `app.js`: Interactive logic, drag-and-drop hooks, polling loop, and API connections.
- `outputs/`: Saved image outputs generated by the queue.
- `queue_manager.db`: SQLite database storing tasks and generation history.
